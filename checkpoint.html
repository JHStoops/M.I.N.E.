2/24/2016 - Checkpoint

I have already implemented most of the basic game logic needed for the core of Minesweeper.
A few things still remain, like disabling the player from revealing more tiles when the game is over,
and recognizing that all the bombs have been flagged to celebrate the player's victory.

Seeing that I'm only half-way through the semester, I plan on implementing additional game logic
which will add to the player's experience of the game. Such game logics will include, but no be limited to,
keeping score, add a player wallet and a store; buy booster items, for example a radar to flag all mines in a
3x3 or 5x5 radius of the user's click, bomb defuser's suit to let player survive a bomb or two, etc.

Main Game Logic Components

    Implemented

  • Generate Grid - Randomly assign bombs, and assign correct numbers to adjacent tiles.
    To test, comment out line 189 in View.js, and uncomment line 190, then start new games to see an uncovered map.
  • Generate Map - Display the player's unexplored view of the map. Available in Classic and Grassy.
    To test, just choose your difficulty and tile type, then press "Start New Game!"
  • Destroy Map - Deletes all information about previous map. Ensures good memory management, and prevents bugs.
  • Event Handlers - Listens for player mouse clicks, and adds functionality to vital buttons (e.g. start new game!)
    To test, left-click on any in-game tile to reveal it, or right-click on a tile to flag it, also notice that the context menu doesn't appear on the map.
  • Flag Tiles - Involves the player right-clicking on any given tile to flag it.
  • Explore Tile - Involves player left-clicking an unexplored tile to reveal it's contents.
  • Explore Empty Tile - recursively opens all adjacent tiles until the empty tiles are surrounded by numbers.
    To test, simply start a new game, and reveal tiles until you find an "empty" tile which will trigger this game logic.
  • Interface functions - Added to View for communicating with the Model. More might be added as needed.

    Not Implemented

  • Detect Success - Celebrate player's victory (later add money to wallet, increase score count, etc.)
  • Detect Failure - Lament player's near death experience (later take money from wallet, increase death count, etc.)
  • Disable Map Clicks - Once the player wins, the map needs to disable any event listeners until a new game has started.
  • First Click is Empty - A player's first click should always be on an empty tile.

Optional Game Logic Components

    Implemented

  • Animated explosion - When you step on a mine, let it do what it does best: Explode!
    To test, accidentally of purposefully click on a mine to witness the explosion up close!

    Not Implemented

  • M.I.N.E. Shop - Capitalism rules! Buy your way to success with better equipment. (disposible items.)
  • Player Wallet - Capitalism rules! Money looks like success, it smells like success, and it tastes like success.
  • Player Statistics - Keep track of wins and losses for each map type, timed records for each map, total money earned/spent, etc.
  • Timer - The bomb defusing training taught you patience, but your inner child wants to see how quickly you can do it!
  • Customized Map - Let the player set the width, height, and mines themselves.
  • Mass Explosion - When you step on a mine, it will trigger all other unflagged mines to explode.

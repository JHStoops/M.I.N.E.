The rules of the game

To win: You must safely flag all mines on the map.
To lose: Step on a mine before flagging it.
Here are some strategies to help you stay alive on the mine field!

You can take on maps of different difficulties:

  • Beginner: 10 mines on a 9x9 map.
  • Intermediate: 40 mines on a 16x16 map.
  • Expert: 99 mines on a 16x30 map.

The types of tiles are

  • - These have yet to be dealt with.
  • - These are explored but so far away from mines that they're no threat.
  • - This means there are two adjacent tiles that have mines. Number varies from 1-8.
  • - Marked as a mine. Doesn't activate when clicked on.
  • - Might be a mine. Activates when clicked on.
  • - If you see this, you've either won or lost.

Preliminary Software Design

Model

Variables

theGame - The game's Model object containing all the information about the gridmap.
{gridRows: new Array(), width: 0, height: 0, mines: 0, wins: 0}

gridRows - A 2D array that stores the Tile values beneath the user's "Unexplored" tiles. By separating the player's map and this grid, it reduces the chance of cheating. Each element contains a string with the class name of the tile, e.g. "empty" or "mine". For example:

gridRows[4] = {"empty", "mines1", "mines2", "mine", "mines4", "mine", ...}

gridRows[5] = {"empty", "empty", "mines3", "mine", "mines5", "mines2", ...}

gridRows[6] = {"mines3", "mines2", "mines2", "mine", "mine", "mine1", ...}

width/height - I could get these values from gridRows's length attributes directly, but using these variables will keep my code simpler and more understadable.

mines - Number of total mines in the player's map. Will be used to show remaining unflagged mines.

wins - A counter for how many mine fields the player has successfully completed.


Functions

getTileValue(row, col) - An interface function for View to get the selected tile's class name.

destroyMap() - An interface function for View to delete the previous map if it exists.

generateGrid(height, width, difficulty) - An interface function for View to create a new mine field.

gameOver(bool) - An interface function for View to tell the Model that the player won (bool == true) or lost (bool == false) the game.

assignMines() - Places mines in random locations, but never in the same location.

numberMines() - Traverses through theGame, figuring out which numbers to put in each tile based on adjacent mines. Skip tile if it's a mine.


View

Variables

Difficulty - Choose difficulty in the form of a radiogroup, "Beginner" being the default.


Functions

init() - Sets event handlers to left/right mouse clicks: disables the right-click context menu, and to appropriately interacts with tiles as they're clicked. Initialized by 'window.onload = init;'

startNewGame() - Uses Model interface functions destroyMap() and generateGrid(height, width, difficulty) to create a new grid, then it runs generateMap() to display the player's map. Activates when player has chosen settings and clicks "Start New Game!"

generateMap() - Each table data element gets a unique ID to make changing the tiles' backgrounds easier by just changing the class. ex: id="r3c6" (row 3, column 6)

reveal() - Tied to an event handler for left-clicks on "unexplored" tiles to reveal its contents. Will call revealEmpty(row, col) if content is "empty", or gameOver(false) if it's a "mine".

revealEmpty(row, col) - Reveal all empty tiles and the surrounding numbers when the player clicks on an empty tile.

markTile() - Any Unexplored tile can be set to "Maybe," "Flagged," and back to "Unexplored" by right-clicking on them.

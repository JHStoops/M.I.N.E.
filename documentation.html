This documentation will follow the same format as the checkpoint from 3/23/2016.
All core game functionality has been implemented. I also went through each function to
ensure that they were in the right file (Model vs. View.) And I made sure that proper

Main Game Logic Components

    Implemented

  • Generate Grid - Randomly assign bombs, and assign correct numbers to adjacent tiles.
    To test, play a few games to see how random they are..
  • Generate Map - Display the player's unexplored view of the map. Available in Classic and Grassy.
    To test, just choose your difficulty and tile type, then press "Start New Game!"
  • Destroy Map - Deletes all information about previous map. Ensures good memory management, and prevents bugs.
  • Event Handlers - Listens for player mouse clicks, and adds functionality to vital buttons (e.g. start new game!)
    To test, left-click on any in-game tile to reveal it, or right-click on a tile to flag it, also notice that the context menu doesn't appear on the map.
  • Flag Tiles - Involves the player right-clicking on any given tile to flag it.
  • Explore Tile - Involves player left-clicking an unexplored tile to reveal it's contents.
  • Explore Empty Tile - recursively opens all adjacent tiles until the empty tiles are surrounded by numbers.
    To test, simply start a new game. The first tile you reveal will be an "empty" tile which will trigger this game logic.
  • Interface functions - Added to View for communicating with the Model. More might be added as needed.
  • Disable Map Clicks - Once the player wins/loses, the map needs to disable any event listeners until a new game has started.
    To test, simply win/lose a game and try left or right clicking on the tiles.
  • First Click is Empty - A player's first click should always be on an empty tile.
    To test, simply start a new game as many times as you want.
  • Detect Success - Congratulate player's victory (add money to wallet, increase win count, etc.)
    To test, look at the side panel under Game Stats. The status will change to "Win!"
  • Detect Failure - Lament player's near death experience (Remove money from wallet, increase loss count, etc.)
    To test, look at the side panel under Game Stats. The status will change to "Coma for X day(s)."
  • Remaining Mines - In the side panel, under Game Stats, you can see thenumber of total mines and mines left.
  • Create Custom Map - Before starting a game, the player can select height and width 9-40 tiles,
    and between 9-400 mines. The equation for max mines is ((height * width) / 2).
    To test, try creating a map that has more than half the tiles as mines: an error message will pop up.

Optional Game Logic Components

    Implemented

  • Animated explosion - When you step on a mine, let it do what it does best: Explode!
    To test, accidentally of purposefully click on a mine to witness the explosion up close!
  • Mass Explosion - When you step on a mine, it will trigger all other unflagged mines to explode.
    To test, accidentally of purposefully click on a mine to witness the array of explosions up close!
  • Customized Map - Let the player set the width, height, and mines themselves.
    To test, chose 'Custom' from the drop-down menu to bring up the input fields. Then play a custom game!
  • Player Wallet - Capitalism rules! Money looks like success, it smells like success, and it tastes like success.
  • Player Statistics - Keep track of wins and losses, timed records for each map, total money earned/spent, etc.
  • Focus Music - To get the extra credit, I added some music with an audio tag.
    To test, simply left-click the play/pause button once. Press again to pause.
  • Use Local Storage - Will take the player's username from Index.html to start a game.
    If the player closes out and returns to Sweeper.html, it'll continue where they left off.
    If the local storage is wiped out and they return to Sweeper.html, it'll start from scratch with the name "Mysterio."
  • Load Game - Lets player load a game from a local JSON file. (currently hardcoded file.)
    To test, click "Load Game" to see the loaded game pop up. (Doesn't work in Chrome!)

    Not Implemented

  • M.I.N.E. Shop - Capitalism rules! Buy your way to success with better equipment. (disposible items.)
  • Timer - The bomb defusing training taught you patience, but your inner child wants to see how quickly you can do it!
  • Save Game - Let user save game in JSON format locally on their computer.
